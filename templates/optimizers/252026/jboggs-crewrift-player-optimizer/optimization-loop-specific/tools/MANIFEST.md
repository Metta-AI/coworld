#### 2. Build a gating multi-seed multi-candidate A/B harness with structured output that shares baseline runs
`loop` · tool: `ab.py`

Run each (policy, seed) pair as a subprocess parsing the score from stdout, accept one baseline plus MANY candidates per invocation (a repeatable --candidate mode sharing the baseline across {baseline,cand1,cand2,...} in one pass), use a bounded THREAD pool (workers only block on subprocesses, no Python CPU; ~3x faster, 22s vs 65s, capped because multi-second per-env init does not parallelize), and exit non-zero when any candidate's mean is strictly worse so it gates automatically. Sequential A/Bs against the substrate baseline and against the full policy each saturate the worker pool on their own baseline and cannot be parallelized as separate invocations, so the shared-baseline multi-candidate mode roughly halves wall-clock per winning cycle and enables cheap parameter sweeps without editing config constants between runs. Emit STRUCTURED machine-readable output (scores, exact commands, git SHA, dirty-tree state, env) with paired-bootstrap CIs, sign-test, trimmed means, and top-N wins/losses; human-only tables force manual transcription and block meta-analysis.

#### 3. Build a reusable in-process trace/diff tool for recurring regression questions; read the score first
`loop` · tool: `ab.py`

For any per-seed regression that persists unchanged across two or more ships, capture and DIFF per-tick per-agent traces (baseline vs current) to find the earliest divergence tick BEFORE forming another hypothesis -- a trace diff is far cheaper than another speculative A/B and repeatedly reveals mechanisms one step removed from the flag (a ~2-hour trace-and-diff tool build plus 20 minutes of analysis resolved a regression open across 15+ speculative experiments). A saved cogames scrimmage replay (--save-replay-dir) contains only static objects and aggregate stats, NOT per-tick frames, so to get per-tick behavior build an in-process trace runner driving mettagrid.Simulator directly (not the 'cogames play' subprocess) so you can grab each AgentPolicy._infos per tick (chosen skill, args, working memory, failures) plus sidecar state the policy never sees (hub inventory, agent positions, deaths); keep the A/B scoring harness as a subprocess for isolation but use the in-process path for trace analysis. It is much slower than a scrimmage, so read the SCORE first -- the failure is often obvious from the regressed score before the trace finishes. Build a reusable general capture+diff tool (skill histograms per role, hub-inventory time series, per-agent distance, death timelines, earliest-divergence tick) once you have three or more pending 'why does seed X regress' questions. When hosted logs 404 or are useless, build a deterministic local replay inspector: resim the BitWorld input+hashes from public artifacts and print final per-player state to cleanly separate 'died early' vs 'lost as crew' vs 'wandered until timeout' (in a temp dir to keep the repo clean) -- but note the tension: a thin resim-and-print inspector is worthwhile while rolling your own full replay PARSER is a dead-end detour, and if the real problem is the player's own tracing is too noisy/untargetable, fix the tracing source instead. Desired-but-unbuilt: ab.py --trace-seed N to auto-capture a trace for one interesting seed alongside the aggregate A/B, and trace_compare_agents.py for per-agent-pair trajectory divergence between two traces on the same seed.

#### 6. Run the A/B harness as one subprocess per (policy, seed); render off, fail-fast, thread-parallel
`loop` · tool: `cogames play`

Run the A/B harness as one subprocess (cogames play) per (policy, seed) pair rather than importing the simulator in-process: this avoids cogames init-ordering concerns and prevents one crashed run from contaminating the rest. Use threads (not processes) since workers block on subprocess I/O; expect ~3x speedup bounded by a ~1.5s serial per-subprocess cogsguard init tail. Fail-fast on any worker exception (cancel remaining work and re-raise) — partial A/B results are worse than none. Force render off in batch runs (-r none) for ~25x speedup (a 6-seed run dropped from ~62s to ~2.5s) because the harness defaults render to gui even without a replay file.

#### 8. Structure the lab notebook as typed, validated, append-only per-file entries; orient against it before any A/B and write the hypothesis file first
`loop` · tool: `experiments/notebook/generate_index.py`

Structure a scripted-policy optimization lab notebook as per-file append-only entries under experiments/notebook/ (experiments/E0XX.md, hypotheses/H0XX.md, tournaments/T0XX.md, submissions/S0XX.md), plus a generated index.json, a backlog.yaml, and a protocol.md — not one monolithic file; never edit old entries, create a NEW file per experiment, and regenerate the committed index in the SAME commit (`uv run python experiments/notebook/generate_index.py` produces index.json and backlog.yaml). backlog.yaml and index.json are GENERATED — never hand-edit; change a hypothesis priority via its markdown front-matter and regenerate. The index validator enforces a fixed verdict vocabulary — use the exact tokens {inconclusive, not-shippable, reject, ship, informational} (write 'reject' not 'rejected') and fixed status enums {open, shipped, rejected, closed, deferred}, with cross-field constraints (a ship-version implies a ship verdict; status:shipped requires shipped_in) so malformed entries fail loudly. Use one experiment (E-number) per run-batch: the same hypothesis question with different arms (a sweep of preload=2, preload=3, then gate variants, all testing 'does preloading work and under what conditions') is ONE E-number, while a different aspect of the same H-number gets a new E-number. #### 9. Feature-flag every version difference behind a byte-identical flags-off substrate, freeze each ship as a snapshot, and trace regressions against the frozen config plus the git SHA
`loop` · **negative result** · tool: `experiments/regression_check.py`

Encapsulate every version-to-version behavioral difference as a feature flag on a single policy class, and guarantee all-flags-off is BYTE-IDENTICAL to the upstream starter policy on fixed seeds. Verify this substrate equivalence automatically (experiments/regression_check.py) as a HARD GATE at the start of every session and before every A/B, never after; it proves the only changed variable is the flag under test. If it fails (non-zero exit) the harness is contaminating the baseline — treat every A/B as invalid until fixed; new subsystems (a memory store) must stay empty and gated when flags are off; compile-time flags (Nim `when defined(...)`) work the same way. Re-run the cheap substrate check whenever a non-flag-gated shared path (perception, interpretation, pipeline wrapper, core navigation, decide.py, pursue_target.py) drifts, because otherwise every 'versus bare substrate' A/B is silently invalidated and the symptom is misattributed to the candidate flag; flag-gated code is safe to refactor as long as the flags-off path is untouched. CRITICAL caveat: the flags-OFF check never exercises flag-gated code, so a broken flags-ON ship build can score 0.00 while the substrate check passes a FALSE NEGATIVE — after modifying shared decision/pathfinding code additionally spot-check the actual flagged ship build by confirming known per-seed scores (seed 42=1.42, seed 1=1.72, seed 7=3.08). A V5 ship break from an uncommitted PursueTargetArgs field mismatch passed the flags-off check unnoticed; a new PolicyConfig field must also be wired into the _build_config() call that passes each ClassVar, or A/B candidates overriding the ClassVar silently run as baseline. Freeze each shipped policy version as a named, byte-stable historical snapshot class (e.g. `AlphaFullV<N>` in experiments/historical_policies.py) in the SAME commit that flips the flag in the shipping policy, enforce that every ship commit adds its snapshot entry, and never delete these classes — without the frozen snapshot, anti-regression tournaments cannot compare against the pre-ship configuration, drift becomes invisible, and a 'current vs latest ancestor' comparison silently reports a meaningless all-ties row; when triaging a new version's regressions, trace AGAINST the frozen class, never the live class (a moving target whose reward delta is non-reproducible). A flag-snapshot freezes only the enable_* flag set, not the shared base-class code (stage programs, skills, navigation, memory, obs parser) it re-runs against, so refactors retroactively move every snapshot together — the only true reproducible anchor for a submitted version is the git SHA recorded at submission (experiments/submissions.json); to reproduce a past version, check out that SHA in a worktree or replay the shipped bundle, not just the flags.

#### 16. Build a stuck watchdog correctly and apply it on every nav path
`loop` · **negative result** · tool: `stuck-watchdog/jiggle-recovery`

A direct steer-toward-target controller with no obstacle awareness can burn an entire episode against the first barrier (5 of 6 crewmates pinned at one wall coordinate for thousands of ticks), so a position-delta watchdog that triggers replan/escape when displacement stays below epsilon for N ticks while a non-noop steer is active is essential and easy to omit. Build it with a frame-count threshold (<=2 unique positions in the last 6 moves, or N near-zero-movement frames). Recover with a deliberate PERPENDICULAR jiggle (e.g. 16 ticks, alternating sides each trigger) plus a path-cache clear -- NOT by retrying the blocked action -- and CRITICALLY keep the intended direction held while adding the perpendicular component, because versions that STOPPED pressing the intended direction just wiggled in place. #### 22. Decode mettagrid territory tokens: here is a global observer-relative token, directionals pack boundary pairs, and two encodings conflict
`loop` · ⚠ _session-derived, unverified_

Verified against compiled mettagrid_c.so (Python only registers metadata). territory:here is a GLOBAL token (is_global=True, packed-location byte 254=GLOBAL_LOCATION, 255=empty/padding); its value is observer-relative: 0=neutral/tie, 1=friendly (observer shares the winning team tag), 2=enemy (norm 3, using NEUTRAL_TERRITORY=0/FRIENDLY=1/ENEMY=2). The four directional tokens (territory:north/south/east/west, norm 9) are SPATIAL, carrying a real Location(row,col) in the 13x13 window, and pack a boundary pair into one byte: value = this_cell*3 + adjacent_cell; decode here=value//3, adjacent=value%3. They are emitted ONLY at cells where territory differs from the named-direction neighbor, so same-on-both-sides values (0,4,8) never appear, the location marks the boundary cell, and tokens are sparse and appear late (~step 11 once the window spans an edge) -- a parser assuming a dense per-step grid sees almost nothing; missing directions default to territory_here. Junctions expose net:cogs/net:clips connectivity SEPARATELY from team:cogs/team:clips ownership: a junction can show team:cogs while having lost its net: tag (disconnected, not projecting territory, not scoring) -- track net: and team: independently.
